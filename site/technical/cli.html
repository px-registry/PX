Technical / CLI

CLI Reference

Every command, every flag.

Installation

# Clone and use directly
git clone https://github.com/px-registry/PX.git
cd PX
node cli.js help
# Or via npx (no clone needed)
npx px help

Requires Node.js v16+. No npm install needed — zero external dependencies.

verify

Run rules against evidence and report results. Two modes: direct verification and replay.

Direct verification

px verify --profile=<profile.json> --evidence=<evidence.json>
FlagRequiredDescription
--profileYesPath to profile JSON file
--evidenceYesPath to evidence JSON file

Replay verification

px verify --manifest=<manifest.json>
FlagRequiredDescription
--manifestYesPath to manifest. Loads bundled-evidence.json and bundled-profile.json from same directory.

pack

Fail-close packaging. All rules must pass. Generates proof pack.

px pack --profile=<file> --evidence=<file> [--recipient=<who>] [--purpose=<why>]
FlagRequiredDescription
--profileYesPath to profile JSON
--evidenceYesPath to evidence JSON
--recipientNoWho the pack is for
--purposeNoWhy the pack was created

Output files

draft-manifest.json — verification record
lens.html — self-contained review view
bundled-evidence.json — evidence snapshot
bundled-profile.json — rule definitions
summary.txt — plain text summary
send-these-files.txt — file checklist for recipients

answer-pack

Questionnaire mode. Mixed results OK. Generates answer rows mapped to CAIQ/SIG frameworks.

px answer-pack --profile=<file> --evidence=<file> [--recipient=<who>] [--purpose=<why>]
FlagRequiredDescription
--profileYesPath to profile JSON (should include caiq_ref/sig_ref)
--evidenceYesPath to evidence JSON
--recipientNoWho the pack is for
--purposeNoWhy the pack was created

Output files

answers.csv — Excel-ready answer rows (UTF-8 BOM)
answers.md — human-readable answers
question-map.csv — rule-to-framework mapping with keywords
exceptions.md — failed rules with compensating control placeholders (only if failures)
lens.html — review view (mixed results supported)
summary.txt — plain text summary
draft-manifest.json — verification record (artifact_kind: answer-pack)
bundled-evidence.json — evidence snapshot
bundled-profile.json — rule definitions
send-these-files.txt — file checklist

check

Collect evidence and verify in one step. Requires the source CLI to be installed.

px check --profile=<file> [-- <collector-args>]
# Examples:
px check --profile=profiles/aws-core-controls-v1.json
px check --profile=profiles/github-org-security-v1.json -- myorg myrepo

Evidence is saved to .px/evidence/ for reuse with pack or answer-pack.

init

Initialize a workspace with sample profiles and evidence.

px init --genesis # Verify PX's own governance files
px init --demo # SOC 2 sample data with one deliberate failure
