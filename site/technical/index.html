Technical

How PX works inside

~2,700 lines. Zero dependencies. SHA-256 + Ed25519.

Architecture

Core

SHA-256 hashing, Ed25519 signing (planned), JSON schema validation, deterministic rule evaluation. Node.js crypto module only — no external packages.

CLI Commands

init generate verify pack answer-pack check

Single entry point (cli.js). Each command is a pure function.

Profiles & Collectors

Profiles are JSON rule sets. Collectors are optional shell scripts for automated evidence collection. Two built-in profiles: AWS Core Controls (10 rules), GitHub Org Security (10 rules).

Verification model

1. Evidence hashing

SHA-256 hash of the raw evidence JSON, computed at load time and recorded in the manifest.

2. Rule evaluation

Each rule: dot-notation path into evidence, expected value, operator (eq/gte/lte). Deterministic.

3. Manifest generation

Evidence hash + profile hash + results + timestamp + seal + recipient + purpose.

4. Replay verification

Receiver hashes bundled evidence, runs bundled rules, compares against manifest claims.

Profile format

// Example rule
{
  "id": "iam.mfa_enforced",
  "description": "MFA is enforced for all IAM users",
  "path": "iam.mfa_enabled",
  "expected": true,
  "caiq_ref": "IAM-02.1",
  "sig_ref": "H.2.1"
}
FieldRequiredDescription
idYesUnique rule identifier
descriptionYesHuman-readable explanation
pathYesDot-notation path into evidence JSON
expectedYesValue to check against
operatorNoeq (default), gte, or lte
caiq_refNoCAIQ framework reference
sig_refNoSIG framework reference
answer_templateNoSuggested answer text for PASS
exception_templateNoException text for FAIL

Seal format

PX-A3F2-2026Q2-PASS-10
PXFixed prefix
A3F2Evidence hash chars 8-11 (uppercased)
2026Q2Year + quarter
PASSPASS or MIXED
10Total rules

CLI quick reference

px verify --profile=<file> --evidence=<file>

Run rules against evidence.

px verify --manifest=<file>

Replay verification against bundled inputs.

px pack --profile=<file> --evidence=<file> [--recipient=<who>] [--purpose=<why>]

Fail-close packaging. All rules must pass.

px answer-pack --profile=<file> --evidence=<file> [--recipient=<who>] [--purpose=<why>]

Questionnaire mode. Mixed results OK.

px check --profile=<file> [-- <args>]

Collect + verify in one step.
